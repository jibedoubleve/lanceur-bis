# Base de données de scripts Lua

Une collection d'exemples de scripts Lua prêts à l'emploi pour les alias Lanceur. Chaque script peut être copié directement dans l'[éditeur Lua](fr/pages/usermanual/4.lua-scripting.md) et adapté à vos besoins.

> **Astuce :** Utilisez la recherche intégrée de votre navigateur (`Ctrl+F`) pour trouver rapidement un script par mot-clé.

---

## Router les paramètres vers différents environnements

**Cas d'usage :** Vous avez un alias unique (ex. l'URL d'une application web) et vous souhaitez ouvrir un environnement différent selon le paramètre que vous tapez.

**Fonctionnement :** Le script vérifie `context.Parameters` et le réécrit vers l'URL d'environnement correspondante. Si aucun paramètre connu n'est fourni, l'alias s'exécute avec ses paramètres d'origine inchangés.

**Exemple :** Un alias nommé `myapp` avec ce script vous permet de taper `myapp dev`, `myapp test` ou `myapp prod` pour ouvrir l'environnement correspondant.

```lua
if context.Parameters == "dev" then
    context.Parameters = "https://dev.myapp.example.com"
    return context
end

if context.Parameters == "test" then
    context.Parameters = "https://test.myapp.example.com"
    return context
end

if context.Parameters == "prod" then
    context.Parameters = "https://prod.myapp.example.com"
    return context
end
```

**Configuration :** Définissez le `FileName` de l'alias sur votre exécutable de navigateur (ex. `firefox.exe`) et collez ce script dans l'éditeur Lua. Ajustez les URL pour correspondre à vos environnements réels.

## Détecter automatiquement la dernière version installée d'une application

**Cas d'usage :** Une application (ex. un IDE) installe chaque version dans un dossier séparé. Vous souhaitez que votre alias lance toujours la version la plus récente sans modifier manuellement le chemin après chaque mise à jour.

**Fonctionnement :** Le script définit une liste de répertoires où l'application peut être installée. Il analyse ces répertoires pour trouver des sous-dossiers correspondant à un motif de nom, compare leurs horodatages de dernière modification, et définit `context.FileName` sur l'exécutable dans le dossier le plus récent.

**Exemple :** Vous avez un IDE qui installe ses versions dans `%LocalAppData%\Programs\MyIDE\` ou `%ProgramFiles%\Vendor\`. Chaque mise à jour crée un nouveau dossier (ex. `MyIDE 2025.1`, `MyIDE 2025.2`). Ce script trouve le plus récent et le lance.

```lua
luanet.load_assembly("System")
luanet.load_assembly("System.IO")

local Environment = luanet.import_type("System.Environment")
local Path = luanet.import_type("System.IO.Path")
local Directory = luanet.import_type("System.IO.Directory")
local DirectoryInfo = luanet.import_type("System.IO.DirectoryInfo")
local SpecialFolder = luanet.import_type("System.Environment+SpecialFolder")

---------------------------------------
-- CONFIGURATION : modifiez ces variables
---------------------------------------
local dirPattern    = "*MyIDE*"            -- motif pour trouver les dossiers d'installation
local exeRelPath    = { "bin", "app.exe" } -- chemin vers l'exécutable depuis le dossier d'installation
local searchPaths   = {                    -- répertoires où l'application peut être installée
    { SpecialFolder.LocalApplicationData, "Programs", "MyIDE" },
    { SpecialFolder.ProgramFiles, "Vendor" }
}
---------------------------------------

-- Collecter tous les sous-répertoires correspondants
local candidateDirs = {}

for _, parts in ipairs(searchPaths) do
    local basePath = Environment.GetFolderPath(parts[1])
    for i = 2, #parts do
        basePath = Path.Combine(basePath, parts[i])
    end

    if Directory.Exists(basePath) then
        local dirs = Directory.GetDirectories(basePath, dirPattern)
        for i = 0, dirs.Length - 1 do
            table.insert(candidateDirs, dirs[i])
        end
    end
end

-- Trouver le répertoire modifié le plus récemment
local latestDir = nil
local latestTime = nil

for _, dirPath in ipairs(candidateDirs) do
    local dirInfo = DirectoryInfo(dirPath)
    local writeTime = dirInfo.LastWriteTime

    if latestTime == nil or writeTime > latestTime then
        latestTime = writeTime
        latestDir = dirInfo
    end
end

-- Mettre à jour l'alias pour pointer vers l'exécutable
if latestDir ~= nil then
    local exePath = latestDir.FullName
    for _, segment in ipairs(exeRelPath) do
        exePath = Path.Combine(exePath, segment)
    end
    context.FileName = exePath
end
```

**Configuration :** Modifiez uniquement les trois variables dans le bloc **CONFIGURATION** en haut :

| Variable     | Rôle                                                                                                   | Exemple                                              |
| ------------ | ------------------------------------------------------------------------------------------------------ | ---------------------------------------------------- |
| `dirPattern` | Motif glob pour trouver les dossiers d'installation                                                    | `"*Rider*"`, `"*Code*"`                              |
| `exeRelPath` | Segments de chemin depuis le dossier d'installation jusqu'à l'exécutable                               | `{ "bin", "rider64.exe" }`                           |
| `searchPaths`| Liste des répertoires de base à analyser (chaque entrée est un dossier spécial + segments de sous-dossiers) | `{ SpecialFolder.ProgramFiles, "JetBrains" }` |
